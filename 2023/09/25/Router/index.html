路由器协议与配置（一）

发表于2023-09-25|更新于2023-09-25|学习笔记
|阅读量:

路由

  • 什么是路由
    路由是在任何网络中选择路径的过程

  • 路由器如何工作
    跟由器根据路由表转发数据包，当数据包到达时，路由器首先在路由表中查找其地址，如果不存在此丢弃此数据包，如果存在，路由器会根据此路由条目的下一跳地址，将数据包转发或移动到网络中的下一个点

  • 路由表
    路由表（routing table）或称路由择域信息库（RIB, Routing Information Base），是一个存储在路由器或者联网计算机中的电子表格（文件）或类数据库。路由表存储着指向特定网络地址的路径（在有些情况下，还记录有路径的路由度量值）。

    • 直连网段：路由器接口直接连接的网段;
    • 非直连网段：路由器接口没有直接连接的网段;

  • 路由表路由条目的生成：

    • 知道如何访问直连网段—-为路由器接口设置IP地址并启用此接口—直连路由
    • 知道如何访问非连网段—-直接网段信息告诉路由—-静态路由
    • 让路由器自己去学习——让路由器运行动态路由协议—-动态路由
    • 直连路由：路由器接口所直连的网段形成的路由信息 （直连网段）
    • 静态路由：由管理员手动为路由器指定的路由信息 （非直连网段）
    • 动态路由：路由器通过路由协议学习到的路由信息 （非直连网段）

静态路由

静态路由一般是在路由器中手动配置的固定路由

    下一跳：表示此路由的下一跳IP地址。指明数据转发的下一个设备。
    下一跳地址一定是相邻路由器的接口地址;

默认路由

默认路由（Default route），是静态路由的一种特殊形式，它属于静态路由中的一种。是对IP数据包中的目的地址找不到存在的其他路由时，路由器所选择的路由。默认路由通常会在出口网络中使用。默认路由容易产生路由环路。
默认路由的目标网络是0.0.0.0/0.0.0.0 下一条地址

路由转发数据包过程




  • 网络层的地址：源IP地址 目标IP地址
  • 数据链路层的地址：源MAC地址 目标MAC地址

数据包经由路由转发时源、目的IP地址及MAC地址变化情况

  • 源IP地址和目标IP地址一致保持不变（NAT为特殊应用，会修改源IP为网关自己外网IP）
  • 源MAC地址和目标MAC地址是变化的，因为发送端开始不知道目的主机的MAC地址，所以每经过一个路由器MAC地址是变化的

动态路由

动态路由协议可以让路由器自动学习到其他路由器的网络，并且网络拓扑发生改变后自动更新路由表。网络管理员只需要配置动态路由协议即可，相比人工指定转发策略，工作量大大减少。

常见路由协议

  • RIP
    路由信息协议（Route Information Protocol） 是内部网关协议IGP中最先得到广泛使用的协议。RIP是一种分布式的基于距离向量的路由选择协议，是因特网的标准协议，其最大优点就是实现简单，开销较小。
  • OSPF
    开放式最短路径优先（Open Shortest Path First）是一个内部网关协议（Interior Gateway Protocol，IGP），用于在单一自治系统（autonomous system,AS）内决策路由。OSPF提出了“区域（area）”的概念，每个区域中所有路由器维护着一个相同的链路状态数据库（LSDB）。OSPF利用所维护的链路状态数据库，通过最短路径优先算法（SPF算法）计算得到路由表。OSPF的收敛速度较快。
  • IS-IS
    中间系统到中间系统（Intermediate System-to-Intermediate System）路由协议和OSPF相同，IS-IS也使用了“区域”的概念，同样也维护着一份链路状态数据库，通过最短生成树算法（SPF）计算出最佳路径。IS-IS的收敛速度较快。集成化IS-IS协议是ISP骨干网上最常用的IGP协议。
  • BGP
    边界网关协议（Border Gateway Protocol）是运行于 TCP 上的一种自治系统的路由协议。 BGP 是唯一一个用来处理像因特网大小的网络的协议，也是唯一能够妥善处理好不相关路由域间的多路连接的协议。

动态路由协议类型

  • 内部网关协议（IGP）：在公司内部运行的协议：RIP/ISIS/OSPF
  • 外部网关协议（EGP）：在公司之间运行的协议：BGP

单臂路由

单臂路由（router-on-a-stick）是指在路由器的一个接口上通过配置子接口的方式，实现原来相互隔离的不同VLAN之间的互联互通。
一个物理接口不能对应所有VLAN,也不能同时设置多个IP地址；
使用物理接口下的子接口，每个子接口对应一个VLAN,单独一个IP地址。
子接口是如何表示的：如：g0/0/0 子接口g0/0/0.1 - 4096

注意

  • 和路由器直接相连的接口（物理接口）不能设置IP
  • 路由器的子接口要记得开启arp广播请求（子接口默认不开启arp）

缺点：

  • 路由器的一个端口要承载所有VLAN的通信流量，将会产生网络通信瓶颈
  • 所有子接口全部依托一个物理接口，存在单点故障，而影响VLAN间通信

VLANIF

VLANIF（Virtual Local Area Network Interface）接口是一种三层的逻辑接口，能实现不同VLAN间，不同网段的用户进行三层互通。每个VLAN对应一个VLANIF接口，在为VLANIF接口配置IP地址后，该接口即可作为本VLAN内用户的缺省网关，对需要跨网段的报文进行基于IP地址的三层转发。

浮动路由

浮动路由指的是配置两条或以上的静态路由，相同的目的地址，不同的下一跳地址，另一条下一跳的优先级低，下一跳链路默认选取链路质量优的（带宽大的）作为主路径，当主路径出现故障时，由带宽较小的备份路径顶替主路径。
作用是解决链路单点故障，保持网络的不中断。

注：静态路由默认优先级60，数值越大（范围1-255），优先级越低。
