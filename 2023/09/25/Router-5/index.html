路由器协议与配置（五）

发表于2023-09-25|更新于2023-09-25|学习笔记
|阅读量:

OSPF

开放式最短路径优先（Open Shortest Path First，OSPF）是一种路由协议，它属于链路状态的内部网关协议（Interior Gateway Protocol，IGP），运作于自治系统内部。

OSPF 协议属于三层协议，协议号为89。是企业内部使用的动态路由协议，适用于大规模网络，采用的是分层设计，分为骨干区域和非骨干区域

AS 自治系统，一个AS 表示一个区域范围，用不同的AS号来区分不同的区分范围

版本

  • OSPFv2：支持IPv4网络
  • OSPFv3：支持IPv6网络

OSPF工作过程

建立邻接关系

  • 建立邻居关系
  • 主/从关系协商、DD报文交换
  • LSDB同步（LSA请求、LSA传输、LSA应答）
邻居关系
建立邻居
  • Down(失效状态)：表示最近没有从邻居收到信息，没有启用OSPF的状态
  • Attempt(尝试状态)：只有在 NBMA 网络环境中，才会出现 Attempt 状态。没有启用OSPF的状态，OSPF 路由器会定期连续发出 Hello 数据包，对端在邻居失效时间间隔（dead interval）超时前仍然没有回复Hello报文。
  • Init(初始状态)：初始化状态，第一次收到对端发来的hello包
  • 2-Way(双向通信状态)：邻居状态，相互间周期发送hello的状态，将进行DR和BDR的选举
数据库交换
  • Exstart(信息交换初始状态)：发送DBD报文，选举主从路由器
  • Exchange(信息交换状态)：向邻居发送DD数据包来描述其完整的链路状态数据库
  • Loading(信息加载状态)：发送LSR报文，请求对方发送缺少的LSA和最新的LSA
  • Full：邻居路由器之间已经完成同步过程，邻居路由器形成完全邻接关系
报文类型
  • hello 周期性发送，用来发现和维持OSPF邻居关系
  • DD 用来描述本地LSDB数据库的摘要信息，用于两台设备进行数据库同步
  • LSR 根据对方的DD报文里，请求对方发送缺少的LSA和最新的LSA
  • LSU 用于向对方发送其所需要的LSA
  • LSACK 用来对收到的LSA进行确认
DR & BDR

指定路由器(DR，Designated Router)和备份指定路由器(BDR，Backup Designated Router)

在广播多路访问网络和NBMA网络上会选举DR和BDR。所有其他路由器只与DR和BDR建立完全邻居关系，其他路由器之间的邻居状态停留在2-Way状态。所有路由器都只将信息发送给DR，由DR将网络链路状态LSA广播出去。

为了避免因DR失效导致的单点故障，多路访问网络上还将选举BDR。DR失效时，BDR将成为DR，由于网络上其他路由器已经和BDR形成了完全邻接关系，因此可以将DR失效对网络的影响降至最低。

路由计算

OSPF采用SPF（Shortest Path First）算法计算路由，可以达到路由快速收敛的目的

区域（Area）

  • 骨干区域（Backbone Area）
    骨干区域，Area 0（点分十进制表示为0.0.0.0），是整个OSPF域的核心区域。所有其他非骨干区域必须和骨干区域直接或间接相连。骨干区域的功能是在不同的非骨干区域之间分发路由信息。
    骨干区域必须是唯一而且连续的，但不需要在物理上连续：它可以使用虚拟连接。

  • 标准区域
    标准区域是最通用的区域，它传输区域内路由，区域间路由和外部路由

  • 末梢区域（Stub Area）
    末梢区域不允许AS外部路由通告（AS External LSA，即类型5 LSA）在其内部进行传播的区域。末梢区域的ABR会通告一条默认路由到该区域内所有路由器，任何发往AS外部网络的数据流都将依据默认路由来转发。

  • 完全末梢区域（Totally Stubby Areas）
    完全末梢区域不允许AS外部路由和区域间汇总路由（Network Summary LSA ，即类3 LSA）在其内部传播。同末梢区域一样，完全末梢区域ABR也会向该区域内所有路由器通告一条默认路由，任何发往其他区域以及AS外部网络的数据流都将依据默认路由来转发。将区域设置为完全末梢区域可以显著的降低路由表条目的数量。

  • 不完全末梢区域（Not-So-Stubby Area，NSSA）
    非纯末梢区域允许引入AS外部路由并且扩散到其他区域，NSSA区域同时保留AS内的STUB区域的特征。为了使NSSA能够引入AS外部路由，OSPF定义了新的LSA类型，即NSSA External LSA（类型7 LSA）。NSSA ASBR将产生类型7 LSA，并在NSSA区域内传播。NSSA ABR将会把类型7 LSA转换成5类LSA，并传播到其他区域。

  • 完全非纯末梢区域（Totally NSSA）
    完全非纯末梢区域除了拥有和NSSA相同的特性外，还不允许类型3 LSA在该区域内部传播。完全非纯末梢区域ABR也会向该区域内所有路由器通告一条默认路由。将区域设置为完全非纯末梢区域也会显著的降低路由表条目的数量。

OSPF路由器类型

  • 内部路由器（Internal Router）
    该类设备的所有接口都属于同一个OSPF区域。

  • 骨干路由器（Backbone Router）
    该类设备至少有一个接口属于骨干区域。一台骨干路由器也可以同时是ABR或ASBR。

  • 区域边界路由器Area Border Router，ABR）
    该类设备可以同时属于两个以上的区域，但其中一个必须是骨干区域，ABR用来连接骨干区域和非骨干区域。区域边界路由器为每一个与之相连的区域维护一份链路状态数据库，因此区域边界路由器需要比内部路由器更多的内存资源和更高性能的处理器。

  • 自治系统边界路由器（Autonomous System Boundary Router，ASBR）
    与其他AS交换路由信息的设备称为ASBR。自治系统边界路由器（ASBR）用来把从其他路由协议（如BGP、EIGRP、其他进程号的OSPF等）学习到的路由以路由重分发的方式注入到OSPF进程中，从而使得整个OSPF域内的路由器都可以学习到这些路由（除了末梢区域内的路由器）。一台ASBR可以是OSPF域内非末梢区域的任何路由器，它可以是内部路由器、区域边界路由器、骨干路由器。

LSA类型

OSPF定义了11种LSA，所有类型的LSA都以一个20字节的通用LSA头部开始。这些不同类型的LSA共同描述了OSPF域内的网络拓扑结构。

  • Router-LSA（Type1）
    路由器LSA：每个设备都会产生，描述了设备的链路状态和开销，在所属的区域内传播。
  • Network-LSA（Type2）
    网络LSA：由DR产生，描述本网段的链路状态，在所属的区域内传播。
  • Summary-LSA（Type3）
    汇总LSA：由ABR产生，描述区域内某个网段的路由，并通告给发布或接收此LSA的非Totally STUB或NSSA区域
  • ASBR Summary-LSA（Type4）
    ASBR汇总LSA：由ABR产生，描述到ASBR的路由，通告给除ASBR所在区域的其他相关区域
  • AS-External-LSA（Type5）
    自治系统外部LSA：由ASBR产生，描述到AS外部的路由，通告到所有的区域（除了STUB区域和NSSA区域）
  • Group-Membership-LSA（Type6）
    组成员LSA：是对标准OSPF的一个扩展，使其支持组播路由功能，扩展后的OSPF称之Multicast Open Shortest Path First（MOSPF）
  • NSSA LSA（Type7）
    NSSA外部LSA：由ASBR产生，描述到AS外部的路由，仅在NSSA区域内传播
  • Link-Local LSA（Type8）
    链路本地LSA：专门用于OSPFv3。链路本地LSA包含了每一个接口的链路本地地址和一个IPv6地址栏表
  • Opaque LSA（Type9/Type10/Type11）提供用于OSPF的扩展的通用机制
    • Type9 LSA：仅在接口所在网段范围内传播。
    • Type10 LSA：在始发该LSA的区域内传播。用于支持TE的LSA。
    • Type11 LSA：在自治域内传播，目前还没有实际应用的例子。

OSPF单区域配置

  • 路由器配置OSPF骨干区域

    进程号 ：同一个设备上可以配置不同的进程号 ，不同的进程号代表不同的程序;不同的设备进程号可以不同

    Area 0：代表的是骨干区域

    network：宣告的是之间接口的网段，表名当前接口参与OSPF 计算，能够接收和发送OSPF信息

  • 查看邻居关系建立信息

    Area ID：表示R1当前建立的邻居关系所在的区域

    Neighbor ID：表示邻居路由器的OSPF Router ID

    State：表示邻居路由器的状态，最终的完美状态是full （完全邻接）-同步数据库

    Router ID：由一个长度为32 bits的字段所定义，是一台路由器在自治系统中的唯一标识，用IPv4地址格式来表示。通过命令行手动配置，如果没有手动配置Router ID，设备会从当前接口的IP地址中自动选取一个作为Router ID

路由汇总

通过ospf的路由汇总，可以减少路由信息，从而减小路由表的规模，提高设备的性能。ospf路由汇总只能在ABR（区域边界路由器）及ASBR（自治系统边界路由器），ABR汇总的是ospf域内的路由，ASBR汇总的是ospf的域外路由，OSPF对于区域内的路由不能做汇总。
