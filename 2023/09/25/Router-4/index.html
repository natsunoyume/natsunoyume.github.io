ACL

访问控制列表ACL（Access Control List）是由一条或多条规则组成，可以根据设定的条件对接口上的数据包进行过滤，允许其通过或丢弃。

分类

类型 编号范围 规则描述
基本ACL 2000～2999 仅匹配源IP地址、分片信息和生效时间段信息来定义规则，不精准
高级ACL 3000～3999 可匹配源IP地址，也可匹配目的IP地址、IP协议类型、ICMP类型、TCP源/目的端口、UDP源/目的端口号、生效时间段等来定义规则，匹配数据更加精准
二层ACL 4000～4999 匹配以太网帧头信息来定义规则，如源MAC地址、目的MAC地址、二层协议类型等。
用户自定义ACL 5000～5999 使用报文头、偏移位置、字符串掩码和用户自定义字符串来定义规则
用户ACL 6000～6031 用户ACL在高级ACL的基础上增加了用户组的配置项，可以实现对不同用户组的流量管控。
基本ACL6 2000～2999 匹配源IPv6地址、分片信息和生效时间段来定义规则。
高级ACL6 3000～3999 匹配源IPv6地址、目的IPv6地址、IPv6协议类型、ICMPv6类型、TCP源/目的端口、UDP源/目的端口号、生效时间段等来定义规则。

组成

  • ACL编号：在创建ACL时可以为其指定编号，不同的编号对应不同类型的ACL
  • 规则：即描述报文匹配条件的判断语句。
  • 规则编号：用于标识ACL规则。可以自行配置规则编号，也可以由系统自动分配。
    ACL规则的编号范围是0～4294967294，所有规则均按照规则编号从小到大进行排序。系统按照规则编号从小到大的顺序，将规则依次与报文匹配。
  • 动作：包括permit/deny两种动作，表示允许/拒绝。
  • 匹配项：源/目的IP地址及其通配符掩码，协议类型，TCP/UDP端口号等
  • 通配符：0表示匹配的位置，1表示不匹配的位置

ACL 的工作原理

  • 使用ACL在匹配数据包的过程中，会按照 rule 的号码从小到大依次匹配
  • 如果能匹配成功，则后续的条目不再进行检查；
  • 如果匹配不成功，则继续匹配下面的 rule ；
  • 如果所有的 rule 都没有匹配成功，则执行ACL最后一个隐含的条目
  • 如果 acl 和 traffic-filter 结合使用，acl最后隐含的条目是：允许所有

确定ACL的IN还是OUT方向

以路由器为中心，数据流进入的方向是IN方向，数据流经过路由器转发出去的方向是OUT方向。

NAT

网络地址转换（Network Address Translation，NAT）
NAT是将IP数据报文头中的IP地址转换为另一个IP地址的过程，主要用于实现内部网络（私有IP地址）访问外部网络（公有IP地址）的功能。

静态NAT

Basic NAT

将内部网络的私有IP地址转换为公有IP地址，IP地址对是一对一的，是一直不变的。不节省公有IP地址

  • 边界设备–配置一条出口路由—默认路由

  • 测试

动态NAT

NO-PAT

NO-PAT属于多对多的地址转换，在路由器上配置一个外网IP地址池，当内部有计算机需要和外部通信时，就从地址池里动态的取出一个外网IP，并将他们的对应关系绑定到NAT表中，通信结束后，这个外网IP才被释放，可供其他内部IP地址转换使用

  • 用ACL 匹配 需要访问外网的 内网网段

  • 配置购买的公有地址的地址池

  • 在外网接口 将公有地址和私有地址做关联

  • 测试
    在外网出口 ，抓包 ，看ping包 ，进行轮询转换

NAPT & PAT

端口地址转换 NAPT(Network Address Port Translation) & PAT(port address Translation)
NAPT方式属于多对一的地址转换，它通过使用“IP地址＋端口号”的形式进行转换，使多个私网用户可共用一个公网IP地址访问外网。

NAPT 与 NAT 的区别在于，NAPT 不仅转换 IP 包中的 IP 地址，还对 IP 包中 TCP 和 UDP 的 Port 进行转换。这使得多台私有网主机利用 1 个 NAT 公共 IP 就可以同时和公共网进行通信。

  • 用ACL 匹配 需要访问外网的 内网网段

  • 配置公有地址的地址池

  • 在外网接口 将公有地址和私有地址做关联

Easy IP的实现原理与NAPT相同，它同时转换 IP 地址和传输层端口。不同的是Easy IP没有地址池的概念，而是将接口地址作为公网IP地址进行NAT。
