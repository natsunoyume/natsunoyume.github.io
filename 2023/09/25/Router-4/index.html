路由器协议与配置（四）

发表于2023-09-25|更新于2023-09-25|学习笔记

ACL

访问控制列表ACL（Access Control List）是由一条或多条规则组成，可以根据设定的条件对接口上的数据包进行过滤，允许其通过或丢弃。

分类

类型 编号范围 规则描述
基本ACL 2000～2999 仅匹配源IP地址、分片信息和生效时间段信息来定义规则，不精准
高级ACL 3000～3999 可匹配源IP地址，也可匹配目的IP地址、IP协议类型、ICMP类型、TCP源/目的端口、UDP源/目的端口号、生效时间段等来定义规则，匹配数据更加精准
二层ACL 4000～4999 匹配以太网帧头信息来定义规则，如源MAC地址、目的MAC地址、二层协议类型等。
用户自定义ACL 5000～5999 使用报文头、偏移位置、字符串掩码和用户自定义字符串来定义规则
用户ACL 6000～6031 用户ACL在高级ACL的基础上增加了用户组的配置项，可以实现对不同用户组的流量管控。
基本ACL6 2000～2999 匹配源IPv6地址、分片信息和生效时间段来定义规则。
高级ACL6 3000～3999 匹配源IPv6地址、目的IPv6地址、IPv6协议类型、ICMPv6类型、TCP源/目的端口、UDP源/目的端口号、生效时间段等来定义规则。

组成

  • ACL编号：在创建ACL时可以为其指定编号，不同的编号对应不同类型的ACL
  • 规则：即描述报文匹配条件的判断语句。
  • 规则编号：用于标识ACL规则。可以自行配置规则编号，也可以由系统自动分配。
    ACL规则的编号范围是0～4294967294，所有规则均按照规则编号从小到大进行排序。系统按照规则编号从小到大的顺序，将规则依次与报文匹配。
  • 动作：包括permit/deny两种动作，表示允许/拒绝。
  • 匹配项：源/目的IP地址及其通配符掩码，协议类型，TCP/UDP端口号等
  • 通配符：0表示匹配的位置，1表示不匹配的位置

[R]acl 2000
[R-acl-basic-2000]rule deny source 192.168.1.0 0.0.0.255


[R]int g0/0/0
[R-GigabitEthernet0/0/0]traffic-filter inbound acl 2000
[R-GigabitEthernet0/0/0]undo traffic-filter inbound

[R]int g0/0/2
[R-GigabitEthernet0/0/2]traffic-filter outbound acl 2000
[R]dis acl all //查看设备上的所有ACL配置
[R]dis traffic-filter applied-record //查看ACL 调用的信息

[R1]ip route-static 192.168.2.0 24 192.168.12.2
[R1]ip route-static 192.168.3.0 24 192.168.12.2
[R2]ip route-static 192.168.1.0 24 192.168.12.1

[R2]acl number 2000
[R2-acl-basic-2000]rule 10 deny source 192.168.1.0 0.0.0.255

[R2]int g0/0/2
[R2-GigabitEthernet0/0/2]traffic-filter outbound acl 2000

[R4]ip route-static 0.0.0.0 0 192.168.12.2
[R6]ip route-static 0.0.0.0 0 192.168.23.2
[R5]ip route-static 192.168.1.0 24 192.168.12.1
[R5]ip route-static 192.168.3.0 24 192.168.23.4

[R4]acl 3000
[R4-acl-adv-3000]rule 5 permit tcp source 192.168.1.0 0.0.0.255 destination 192.168.3.0 0.0.0.255 destination-port eq 80
//规则5 允许来自1.0网段的数据访问3.0网段的基于TCP协议的目标端口为80的服务
[R4-acl-adv-3000]rule 10 permit ip source 192.168.1.0 0.0.0.255 destination 192.168.2.0 0.0.0.255
//规则10 允许来自1.0网段的数据访问3.0网段的任何服务
[R4-acl-adv-3000]rule 15 deny ip source 192.168.1.0 0.0.0.255 destination any
//规则15 拒绝来自1.0 网段的数据访问 其他任何网络
[R4]int g0/0/1
[R4-GigabitEthernet0/0/1]traffic-filter inbound acl 3000 //在数据源设备近的端口应用acl3000

[R4]acl 3000
[R4-acl-adv-3000]rule 5 permit icmp source 192.168.1.0 0.0.0.255 destination 192.168.3.0 0.0.0.255
[R4-acl-adv-3000]rule 10 permit ip source 192.168.1.0 0.0.0.255 destination 192.168.2.0 0.0.0.255
[R4-acl-adv-3000]rule 15 deny ip source 192.168.1.0 0.0.0.255 destination any

[R4]int g0/0/1
[R4-GigabitEthernet0/0/1]traffic-filter inbound acl 3000

[R5]telnet server enable
[R5]user-interface vty 0 4
[R5-ui-vty0-4]authentication-mode password
Please configure the login password (maximum length 16):HCIE

[R6]telnet server enable
[R6]user-interface vty 0 4
[R6-ui-vty0-4]authentication-mode aaa
[R6]aaa //进入AAA视图
[R6-aaa]local-user ntd password cipher hcip //创建用户、密码
[R6-aaa]local-user ntd service-type telnet //设置用户的服务类型
[R6-aaa]local-user ntd privilege level 15 //设置用户的权限级别


[R5]acl 2001
[R5-acl-basic-2001]rule permit source 192.168.1.254 0.0.0.0

[R5]user-interface vty 0 4
[R5-ui-vty0-4]acl 2001 inbound

telnet 192.168.12.2 //默认使用出接口12.1 登录不上远端设备
Error: Can’t connect to the remote host

telnet -a 192.168.1.254 192.168.12.2 //挂源制定访问目标使用的源地址
Password: HCIE


[R6]acl 2000 //配置ACL2000列表
[R6-acl-basic-2000]rule deny source 192.168.1.254 0.0.0.0 //拒绝来自1.254 发来的数据
[R6-acl-basic-2000]rule deny source 192.168.12.1 0.0.0.0 //拒绝来自12.1 发来的数据
[R6-acl-basic-2000]rule permit source any //允许所有的数据

[R6]user-interface vty 0 4
[R6-ui-vty0-4]acl 2000 inbound //在VTY 接口下调用ACL

telnet 192.168.23.4 //不能登录
telnet -a 192.168.1.254 192.168.23.4 //不能登录
telnet 192.168.23.4 //能登录

[R1]ip route-static 0.0.0.0 0.0.0.0 200.1.1.9

[R1]int g0/0/1
[R1-GigabitEthernet0/0/1]nat static global 200.1.1.2 inside 192.168.1.1 //将需要访问外网的私有地址和购买的公有地址做转换


[R1]dis nat static //查看NAT静态转换表
[R1-GigabitEthernet0/0/1]undo nat static global 200.1.1.2 inside 192.168.1.1 //删除

[R1]acl 2000
[R1-acl-basic-2000]rule permit source 192.168.1.0 0.0.0.255

[R1]nat address-group 1 200.1.1.2 200.1.1.6

[R1]int g0/0/1
[R1-GigabitEthernet0/0/1]nat outbound 2000 address-group 1 no-pat

[R1]acl 2000
[R1-acl-basic-2000]rule permit source 192.168.1.0 0.0.0.255

[R1]nat address-group 1 200.1.1.2 200.1.1.6

[R1-GigabitEthernet0/0/1]nat outbound 2000 address-group 1

[R1]dis nat address-group verbose //查看地址池详细信息

[R1]dis nat session all //查看NAT 回话表
